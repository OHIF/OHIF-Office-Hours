* **Primary bottleneck: network constraints (slow WiFi)**

  * Bandwidth limitations are often the dominant factor, especially with:

    * Multiple simultaneous viewports (~6)
    * Segmentation-heavy workflows
    * Large volumetric datasets (PET/CT/MR)
  * Even with optimizations, slow connections (e.g., ~10–20 Mbps) significantly impact load times and interactivity

* **HTJ2K considerations**

  * Progressive HTJ2K is beneficial and already implemented
  * Helps with:

    * Faster initial image display
    * Incremental refinement of image quality
  * However, decompression cost and browser-side processing can still impact performance

* **Segmentation impact**

  * Segmentation is one of the most expensive operations:

    * Memory usage increases significantly
    * Worker usage (e.g., surface generation, statistics) adds CPU overhead
  * Multiple segmented viewports multiply the cost
  * Recommendations:

    * Limit number of simultaneously active segmentations
    * Consider alternative representations (e.g., label maps vs surfaces)

* **Viewport scaling challenges**

  * ~6 simultaneous segmented viewports is a heavy workload
  * Each viewport may:

    * Trigger independent data fetches
    * Require separate rendering pipelines
  * Optimization strategies:

    * Lazy load or defer non-visible viewports
    * Reduce resolution or decimate volumes for secondary viewports

* **MIP and 3D rendering**

  * MIP and 3D rotations introduce additional GPU/CPU load
  * Restricting rotation (left/right only) can reduce computational complexity

* **Metadata and request optimization (already implemented)**

  * Off-main-thread metadata fetching improves UI responsiveness
  * Workers for metadata batching reduce blocking
  * Concurrent request limits help prevent network saturation
  * These are strong baseline optimizations

* **Additional recommended optimizations**

  * Introduce caching layers for metadata and image data
  * Compress metadata payloads where possible
  * Use series-level queries instead of full metadata loads when feasible
  * Prioritize loading of visible viewport data first
  * Consider splitting large segmentation datasets into smaller logical groups
